# Text Elements
Análisis Semántico ^NULUiVsh

La semántica de un lenguaje es el conjunto de reglas formales que definen el significado de los
programas sintácticamente correctos. ^Zj2XbkIt

Objetivos del Analizador Semántico ^VzYafkkz

Si se produce un error semántico, es notificado al manejador de errores.
Veamos escenarios de ejemplo:
    
    1. Definición de variables y reglas de ámbito (scope)

    2. Unicidad: entidades que deben ser únicas (varDef en el mismo scope)

    3. Enlaces: cuando el uso de una entidad deber vincularse a otra construcción

    4. Comprobaciones necesarias (L-Value, sentencias Return, break y continue, etc)

    5. Comprobaciones dinámicas (en tiempo de ejecución como el acceso a arrays)

    6. Comprobación de tipos (inferencia de tipos y comprobación de expresiones válidas)


Un analizador semántico verifica las reglas semánticas recorriendo el AST y añadiendo información adicional
como puede ser los tipos. A este nuevo AST se le denomina AST decorado.

Veamos un ejemplo rápido de como quedaría el AST después de decorarlo con nueva información:
 ^AvWAmnHW

int a;
double b;
b = (a + 1) * ( b - 8.3); ^ZrkYnb3B

- REGLAS SEMANTICAS DE LA GRAMATICA ATRIBUIDA (R) ^E7DRzuf8

Las AG añaden atributos a las construcciones sintácticas de un lenguaje, siendo estos atributos una propiedad
de una construcción sintáctica. Para cada atributo, su dominio debe ser especificado (por ejemplo, type debe
ser int, char, double...).

Las reglas semánticas en una AG especifican cómo se calcula el atributo de un símbolo a partir de los atributos
de otros símbolos en la misma producción.


Una gramática atribuida es una tripleta AG = {G, A, R}, dónde:
    
    - G es una gramática libre de contexto G = (Vn, Vt, P, S)

    - A es un conjunto finito de atributos

    - R es un conjunto finito de reglas ^BbNThWvM

Evaluación de Gramáticas Atribuidas ^bEjPrsX0

La evaluación de una AG es el cálculo de todos los atributos de un programa dado. Precia calcular el orden de 
ejecución de las reglas semánticas.

Por lo tanto el algoritmo general para la evaluación de AGs es:

    1. Construir el árbol de sintaxis para un programa específico

    2. Crear el grafo de dependencias de atributos

    3. Calcular el ordenamiento topológico del grafo dirigido

    4. Ejecutar las reglas semánticas, siguiendo dicho ordenamiento topológico


Veamos un ejemplo: ^isRz3oV6

(1) S  A B B.i = A.s * 2 ^Zb1CWFEm

(2) A  1 A.s = 1 ^hlDhG0jE

(3) B-> 2 B.s = 2 + B.i ^y72Bohq2

class Arithmetic implements Expression {
    Expression expr1, expr2;
    String oper;
    boolean lValue; //getters y setters

    R accept<R, P> (Visitor<R,P> v, P param) {
        return v.visit(this, param);
    }
}

// Esto para el resto de nodos del AST (implementar el accept) ^kqkgYTPJ

class ValueVisitor implements Visitor<void, void> {
    
    @Override
    public Void visit(Arithmetic a, Void p) {
        a.setLValue(false); // Por defecto en AbstractExpression
        a.expr1.accept(this, p);
        a.expr2.accept(this, p);
        return null;
    }

    @Override
    public Void visit(Assignment a, Void p) {
        a.expr1.accept(this, p);
        a.expr2.accept(this, p);
        a.setLValue(a.expr1.getLValue()); // IMPORTANTE EL ORDEN DE DEPENDENCIAS
        return null;
    }

    // Asi para el resto
}  ^gbwaldq3

Comprobación de tipos ^M5wS26Wp

El analizador semántico comprueba si programas sintácticamente válidos cumplen las reglas semánticas del lenguaje.
Si la regla no se cumple se produce un error de tipo.

Es buena práctica separar la fase de comprobación de tipos y la de identificación de símbolos, siendo esta última la
que se realiza antes de la comprobación de tipos.
